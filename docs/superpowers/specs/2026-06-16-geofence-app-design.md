# ジオフェンス警告アプリ 設計書

## 概要・目的

`toyosu.geojson` に定義されたポリゴンをジオフェンスとして使い、視覚障害者(弱視を含む)向けの安全確認アプリを作る。スマートフォンのGPS(位置情報)とコンパス(方位センサー)を使い、利用者が安全エリアの境界に近づいたら「注意」「警告」を、エリア外に出たら「危険」を、画面表示とビープ音で伝える。

豊洲以外のエリアでも使えるよう、ジオフェンスのポリゴン(geojsonファイル)は差し替え可能にする。

## アーキテクチャ

- プレーンHTML/CSS/JS(ビルド不要)の単一ページアプリ
- `config.js` に定義した `GEOJSON_PATH`(既定: `./toyosu.geojson`)からfetchでポリゴンを読み込む
- ブラウザのGeolocation API(`watchPosition`)で現在地を継続取得
- ブラウザのDeviceOrientationEvent APIで端末の向き(コンパス方位)を取得
- Geolocation/DeviceOrientationの許可取得はユーザー操作(「開始」ボタン)から行う(iOS Safariの仕様上、ユーザージェスチャーが必要)

### ファイル構成

```
toyosu-blind/
├── index.html          # UI(状態表示画面)
├── style.css
├── app.js              # 状態管理、ジオロケーション/方位センサー制御、UI更新
├── geo-utils.js         # 内外判定・距離計算・方位計算のユーティリティ
├── audio.js             # Web Audio APIによるビープ音生成
├── config.js            # 閾値定数、GEOJSON_PATHなどの設定値
└── toyosu.geojson       # ジオフェンス用ポリゴン(差し替え可能)
```

## UI(画面表示)

地図表示は行わず、シンプルな状態表示画面とする(低スペック端末でも軽量、情報を絞って直感的に把握できる)。

- 起動直後: 「開始」ボタンのみの画面。タップで位置情報・方位センサーへのアクセス許可をリクエストし、Web Audio APIのオーディオコンテキストをアンロックする
- 開始後、以下を大きな文字・色で表示:
  - **状態テキスト**: 「安全」「注意」「警告」「危険」
  - **最も近い境界までの距離**(m)
  - **境界方向への矢印**: コンパス方位を使い、画面上で矢印を回転させて最近接の境界線への方向を示す
  - 「危険」状態では画面全体を赤くする
  - GPSの`accuracy`が閾値を超える場合は「GPS精度低下中」を別途表示

## 警告レベル(4状態)

設定定数として `config.js` にまとめ、コード編集だけで調整可能にする:

```js
const CAUTION_M = 20;       // 内側、境界まで20m以下 → 注意
const WARNING_M = 8;        // 内側、境界まで8m以下 → 警告
const DANGER_OUTSIDE_M = 3; // 外側、境界まで3mを超えたら → 危険
const ACCURACY_WARN_M = 20; // GPS accuracyがこれを超えたら精度低下表示
```

| 状態 | 条件 | 表示色 | 音 |
|---|---|---|---|
| 安全 | 内側、境界まで > `CAUTION_M` | 緑 | 無音 |
| 注意 | 内側、境界まで ≦ `CAUTION_M` | 黄 | 低速ビープ(例: 2秒間隔) |
| 警告 | 内側、境界まで ≦ `WARNING_M` / または外側、境界まで ≦ `DANGER_OUTSIDE_M` | オレンジ | 中速ビープ(例: 0.7秒間隔) |
| 危険 | 外側、境界まで > `DANGER_OUTSIDE_M` | 赤(画面全体) | 連続アラート音(途切れないトーン) |

境界をまたいだ直後(外側 ≦ `DANGER_OUTSIDE_M`)はまだ「警告」のままとし、さらに離れたら初めて「危険」になる。これにより、境界を一瞬踏み越えただけで最強の警告が鳴ることを避ける。

注意/警告/危険でビープ音の周波数(音色)も変える(危険時は高めで耳につく音にする)。

## ジオフェンス判定ロジック(`geo-utils.js`)

- 現在地付近を基準に、緯度経度を簡易等距離円筒図法でメートル単位の平面座標に変換する(対象エリアの規模であれば十分な精度)
- **内外判定**: Ray castingアルゴリズムでポリゴンの内側/外側を判定
- **境界までの距離**: ポリゴンの各辺(線分)について現在地からの最短距離を計算し、最小値(最も近い辺までの距離)を採用
- **方向**: 最近接点への方位角(ベアリング)を計算し、DeviceOrientationEventで得た端末の向きとの相対角度から、画面上の矢印の回転角を決定

## GPS精度の扱い

- `position.coords.accuracy` が `ACCURACY_WARN_M` を超える場合、状態判定自体は取得した座標(中央値)で計算しつつ、画面・音声で「GPS精度低下中」であることを別途利用者に伝える

## テスト・開発モード

実際に豊洲(または対象エリア)に行かなくても動作確認できるよう、デバッグ用の機能を用意する:

- 画面操作(またはURLパラメータ)で緯度経度を手動入力し、実際のGPS位置情報の代わりに使えるモード
- 方位もスライダーなどで手動指定できるようにする

## ホスティング・デプロイ

- ローカル開発: `localhost` はブラウザの特例でHTTPS不要のままGeolocation/DeviceOrientation APIが使える。簡易HTTPサーバ(例: `npx serve`)で動作確認する
- 本番公開: GitHub Pagesなど無料のHTTPSホスティングを想定。現在このディレクトリはgitリポジトリではないため、必要に応じて`git init`から行う

## ポリゴンの差し替え

- 豊洲以外のエリアで使う場合は、`config.js` の `GEOJSON_PATH` を変更し、対応する1つのFeature/Polygonを持つgeojsonファイルに置き換えるだけで動作する
